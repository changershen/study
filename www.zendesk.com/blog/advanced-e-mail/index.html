Advanced E-mail Features

by Alexander Aghassipour on September 30, 2007

Help desk end-users can submit a new ticket by sending an email to support@accountname.zendesk.com, where accountname is the name of your help desk. In addition, when a ticket is updated the requester and assignee of the ticket receives the updates per mail. This is called a notification. By replying to the notification, the ticket is commented accordingly. Basically you just need to include the ticket ID in the mail subject to have your e-mail appended to the relevant ticket. Notifications are sent via triggers and are fully configurable. Adjust and create notifications that fits your business rules.

A less known feature lets operators set ticket priorities via e-mail. Properties can be set by stating the relevant commands in the subject of your e-mail. This also serves as a high-level integration method.

Operators in your help desk can set properties when working with tickets via mail. Properties are set in the subject of the mail, and are enclosed by double brackets. Thus, by including “{{assignee:john priority:high}}” in the subject, you assign the ticket to “John”, and set the priority to high. If more than one operator named “John” exists in your help desk, the property will not be set.

Ticket properties that can be set via mail

  • status – valid values are open, pending and solved
  • requester – only relevant when creating a ticket. Sets the requester of the ticket. The “from” email address is always set to the submitter.
  • group – assigns the ticket to a group.
  • assignee – assigns the ticket to an operator.
  • priority – sets the priority of the ticket. Valid values are low, normal, high and urgent.
  • ticket_type – valid values are incident, question and problem.

Notice that any invalid property settings will discard all property settings for the mail operation (e.g. designating an assignee without designating a group, when a help desk has more than one group).

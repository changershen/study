Tickler PLUS

Checklists you can create, save and reuse in any ticket

(16)
Buy

$1.95 per agent, per month

Overview

✓ Save checklists for reuse in other tickets ✓ Append saved lists to existing list ✓ Extra statuses: pending and skipped

With Tickler PLUS you can not only create to-do checklists in your tickets, but you can also save them like templates for reuse in other tickets. You have the choice to append any saved checklist to the existing checklist rather than replace it. There is no reusing of checklists available in the FREE version of Tickler.

Tickler PLUS also introduces two new statuses, "Pending" and "Skipped". The FREE version of Tickler only has the statuses "To-do" and "Done".

If your agents are following defined processes to handle certain types of tickets/issues, then Tickler PLUS is great for setting up checklists that step your agents through the processes required to resolve these tickets. Admin agents can create the checklists as "Shared lists" that can then be used by all agents. Non-admin agents can create "Personal lists" that they can reuse.

Other features

  • Predifine processes and workflows
  • Drag and drop to re-order tasks
  • Easy to configure

Read more at Tickler's home page.

Please refer to the user guide for how to use Tickler PLUS.

Tickler is brought to you by Lovestock & Leaf.

www.lovestockleaf.com

After installing Tickler PLUS you will need to set up a custom ticket field for Tickler PLUS to store its data. Don't worry, Tickler PLUS will hide this field away so your agents won't know it's there.

As an administrator you:

  • Click on the "Admin" (cog icon).
  • Click on "Ticket Fields" item in the "Manage" section on the left.
  • Click on "add custom field" up on the top right of this screen.
  • Click on "select »" for the "Multi-line text".
  • In the "For agents" give the field a name (we suggest "LLData_Tickler").
  • Click on "Add field".
  • At the top there will be a notification saying that the field was created. Click on the field name.
  • At the top of this screen it has "Custom field ID:". Copy that ID down.
  • Click on the "Manage" item in the "Apps" section on the left.
  • Click on the cog icon to the right of Tickler PLUS and click on "Change Settings".
  • Enter the ID from the field you created into the "Tickler data field" setting.
  • Click on "Update".
  • Click on "User Fields" item in the "Manage" section on the left.
  • Click on the "Multi-line text" field in the right list.
  • In the "Field title shown to agents" you must use the name "LLData_Tickler".
  • Click on "Create field".
  • Once the field has been created, you will need to refresh your Zendesk. So click the refresh button in your browser.

IMPORTANT: If you're on Zendesk's Enterprise plan and you're using Ticket Forms, please make sure that the data field you created is added to every ticket form.

Configuration Options

Use pending status? If this is ticked then the pending status will be one of the status options. If this is not ticked then you will not be able to set tasks as pending. The default is ticked.

Use regular status order? If this is ticked, then the order of the status options is: None, Pending, Done. If this is not ticked then the order of the status options is: None, Done, Pending. The default is not ticked.

  • Tickler data field This is the ID of the custom ticket field which is used to store the Tickler PLUS data. Note that this field can be shared with the existing Tickler FREE app, but not with the Tickler PRO app.

Enable role restrictions?: Allows you to choose which agent roles that the app will be available for. If this is not ticked then all agents will be able to use the app.

Upgrading from Tickler FREE to Tickler PLUS

If you've been using Tickler FREE and wish to migrate the lists into Tickler PLUS that have already been created in FREE, then you should install Tickler PLUS and set the Tickler data field setting to the same field that Tickler FREE was using. Then disable (or delete) the Tickler FREE app.

If you don't wish to retain the existing checklists, then you can simply use a new data field, and delete the old Tickler FREE one if you're totally sure you don't need it.
